# Module 4: Troubleshooting Common Issues

**Estimated time:** 60 minutes

**Purpose:** Keyboards that were working can stop working — a Windows update, a new application, an accidental setting change. This module gives you a way to think through problems quickly instead of guessing, so you can help someone else stay calm when it happens to them.

## Learning objectives

By the end of this module, you will be able to:
- Sort a keyboard problem into one of four common categories
- Apply the matching quick fix for each category
- Identify where to find further help when a problem doesn't fit a known pattern

## Connect

"It worked yesterday — now it's gone."

You've probably heard something like this. Keyboards are reliable when they're set up well, but small things can disrupt them — a Windows update, a new application, clicking the wrong setting by accident.

This module won't make problems disappear. But it will help you think through them quickly, so you can stay calm and figure things out when a community member comes to you frustrated.

## Content

### Start here: what kind of problem is it?

Most keyboard problems are one of four types. Figuring out which one saves a lot of time.

- **Type 1** — The keyboard isn't showing up. The Keyman icon is missing, or the keyboard isn't in the list.
- **Type 2** — The keyboard is active but the wrong characters appear. You selected the right keyboard, but the output is wrong.
- **Type 3** — Switching doesn't work. The keyboard works on its own, but you can't switch to it properly.
- **Type 4** — The characters look wrong. You're typing the right characters, but they appear as boxes or strange symbols.

Each type has a different fix. Let's go through them.

### Type 1: Keyboard not showing up

If the Keyman icon isn't in the system tray, Keyman might not be running. Search for Keyman in the Start Menu and open it. This should bring it back.

If the icon is there but your keyboard isn't listed, open Keyman Configuration and check Keyboard Layouts. The keyboard might have been removed accidentally, or a Keyman update may have reset things. Just reinstall the keyboard.

### Type 2: Wrong characters appearing

This almost always means the wrong keyboard is selected. Look at the system tray icon — it shows which keyboard is active. Click it and switch to the correct one.

If you have the right keyboard selected but still get wrong characters, open the on-screen keyboard (`Shift + Alt + K` — Keyman's "Show On Screen Keyboard Pane" hotkey). Compare what the keys show with what you're actually getting. If they don't match, another keyboard might be overriding Keyman.

**TIP** Some programs have their own language settings that can override the system. If one specific program is causing trouble, check its language or input settings separately.

### Type 3: Switching doesn't work

`Win + Space` only works if Windows has more than one language installed. Go to **Windows Settings → Time & Language → Language & Region** and check. If only one language is there, add another.

If languages are there but switching still doesn't work, set a custom hotkey in Keyman Configuration's Hotkeys panel (see Module 3). This gives you a direct shortcut that doesn't go through Windows at all.

**NOTE** If Keyman's own `Left Alt + Shift` language-switcher hotkey seems to do nothing, remember the multi-monitor quirk from Module 3 — it switches on Screen 1 regardless of which screen you're actually working on. Check the other screen before assuming it's broken.

### Type 4: Characters look wrong (boxes or strange symbols)

This is usually a font problem, not a keyboard problem. The keyboard is working correctly, but the program is using a font that doesn't include those characters. The fix is to change to a font that supports your language's script.

Noto fonts (from Google Fonts) cover a huge range of scripts and are free. SIL also makes fonts for many minority language scripts — an archive of the original scripts.sil.org resources is available at [scripts.sil.org](https://scripts.sil.org/) (note: this is an archive, not an actively-maintained current site, so treat it as a reference rather than a live support channel).

**TIP** In Word, it helps to set the font before you start typing in the new language. Changing the font after the fact sometimes works, but starting with the right font is cleaner.

### Where to get more help

You don't have to figure everything out alone. These are good places to go:

- [community.software.sil.org](https://community.software.sil.org/) — the Language Software Community site; for Keyman specifically, you'll need to scroll down the page to find the relevant area, it isn't at the top
- [keyman.com/en/support](https://keyman.com/en/support/) — official documentation and support articles
- Your LangTech Consultant network — colleagues have probably seen the same problem

## Challenge

Reading about troubleshooting is useful. Working through a real problem is better.

**✏️ Scenario:** The Keyman icon is visible. Your keyboard is listed. You click it, then try to type in Word — but the wrong characters appear. The on-screen keyboard shows the right keys. What would you check? Write down each step you'd take.

There's no single correct answer. The point is to practice the thinking, not find one specific solution. If you want to go further, try deliberately breaking your setup — switch to the wrong keyboard, or change a font — and then troubleshoot back to working.

## Change

This is the last module. It's a good moment to look back at where you started.

**✏️ Final reflection:** Go back to what you wrote in Module 1 — the typing or display problem you described. If you saw that same problem today, what would you do? Be specific.

If you can answer that clearly, you've reached the goal. You can now install, configure, and troubleshoot Keyman for Windows — and help others do the same.
