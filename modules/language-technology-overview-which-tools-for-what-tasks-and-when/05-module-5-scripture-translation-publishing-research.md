### Content (40 minutes)
### Primary Translation Tools
Play the video Scripture Translation Tools
**Paratext** - Industry standard for Bible translation
- FREE for registered Bible translators
- Stores data in USFM (Universal Standard Format Markers)
- World-class collaboration features (teams work together efficiently)
- Extensive translation resources for vetted translators
- Powerful Scripture checking tools (consistency, error detection)
- Can handle adaptation and back translation
**Adapt It** - First-draft acceleration
- Word-by-word Bible translation (handles phrases too)
- Best for quick first drafts
- Requires: good source text, linguistically related languages, translator knows both languages well
- Excellent for back translations (translating back to consultant’s language for accuracy checking)
**Common workflow:** Create first draft in Adapt It → Refine in Paratext toward final translation
**FLExTrans** - Machine translation for low-resource languages
- Integrates FLEx (lexicon management) with Apertium (open-source machine translation)
- Designed specifically for languages with limited resources
- Makes Bible translation more efficient and intuitive
**Scripture Forge** - Web-based collaborative platform
- Teams draft and edit translations together online
- No version reconciliation headaches—everyone stays synchronized
- AI-generated preliminary drafts from existing Bible translations (reduces workload, speeds process)
- Gives translators a head start
**Paratext Lite** - Mobile companion
- Runs on Android tablets/phones<br>PTXPrint- Doesn’t have full Paratext capabilities
- Enables basic functions on portable devices
- Read, review, or revise Scripture on the go
**Output:** Scripture in the target language
### Checking Tools
**Paratext** (built-in checking)
- Powerful consistency checks
- Error detection
- Bible translation quality assurance
**Transcelerator** - Paratext plugin for comprehension checking
- Helps translate comprehension questions into target language
- Semi-automates the process of preparing checking questions
- If you’ve wondered how to translate all those English checking questions—this is your tool
**Scripture Forge - Community Checking Tool** - Broader feedback
- Facilitates Scripture checking with wider community input
- Works with Paratext
- Delivers feedback to translation teams
**Output:** Checked Scripture ready for publication
### Scripture Publishing Tools
**PTXprint** - High-quality PDF creation
- Paratext companion for trial publications
- Creates professional PDFs for Scripture testing
- User-friendly interface (anyone can produce Scripture for local testing)
- Highly configurable output options
- Built on PrintDraft technology (significantly enhanced)
- Mission: “Publish early and publish often” for iterative improvement through community feedback
**Publishing Assistant** - Professional typesetting workflow
- Supports typesetters preparing Paratext USFM for publication
- Interface for entering Bible layout specifications
- Automates nearly all Adobe InDesign layout operations
- Eliminates manual typesetting tasks
**InDesign** - Industry-standard layout software
- Main program SIL International Publishing Services uses
- Professional-grade typesetting
**XeTeX** - Alternative typesetting program
- Another option for Scripture layout
- Open-source solution
**Scripture App Builder** - Android Bible apps
- Takes Paratext files, DBL files, or formatted Word documents
- Creates fully functional Bible apps for Android
- Minutes from Paratext file to functioning Bible app
- Puts Scripture on phones and tablets
**Story Producer** - Oral translation video creation
- Enables local speakers to make oral translations
- Works with illustrated Bible stories and Bloom books
- Creates shareable videos with essentially one tap
- Phone-to-phone sharing in communities
- All on basic Android phones
**Output:** Publishable Scripture in multiple formats (print, PDF, mobile apps, video)
### Oral Translation Tools
**Render** - Oral Bible translation software
- Speaker hears Scripture passage in language they understand
- Translates and records in their language
- Peer review, community testing, consultant checking
- Output: recorded Scripture for listening
**Audio Project Manager** - Speech audio file management
- Web and desktop application
- Manages oral Bible translation, drafting, storying, transcription projects
- Customizable workflows
- User-friendly for minimal computer skills
**One Story Editor (OSE)** - Oral story management
- Documents, organizes, and consultant-checks oral Bible story transcriptions
- For storytelling teams what Paratext is for translators
**HearThis** - Easy Bible recording
- Do-it-yourself Scripture audio recording
- Automatically finds Bible translations
- Walks through recording one section at a time
**Glyssen** - Dramatized audio script preparation
- Produces scripts for high-quality dramatized Scripture recording
- Identifies all direct speech and speakers
- Guides casting and role assignment
- Prepares complete recording scripts
**Purpose:** Making Scripture accessible to oral preference and non-literate communities
### Scripture Research Resources
**Enhanced Resources (in Paratext)** - Translation helps
- Enables Bible translators without Greek/Hebrew to access underlying meaning
- Powerful research, drafting, and checking tools
- Biblical terms linked with resources
**BART** - Interactive source language Bible
- Bible translation research tool
- Access to original Greek and Hebrew Scripture texts
- Helps translators with limited Hebrew/Greek knowledge recognize source text features
**Logos Translator’s Workplace** - Reference library
- Foundational collection for Bible translation
- Bibles, Greek/Hebrew texts, dictionaries, commentaries
- Bible translation handbooks, articles, reference materials
### Challenge (30 minutes)
**Tool Selection Matrix:**
For each scenario, identify the BEST tool and explain why:
**Scenario 1:** A team wants to translate 1 Corinthians. They have a good translation in a related language. They need a rough draft quickly to start community testing.
**Scenario 2:** A translation team of 5 people works in different villages. They need to collaborate without constantly emailing files back and forth. They want AI assistance for initial Scripture drafts.
**Scenario 3:** A consultant needs to check the accuracy of a Bible translation, but doesn’t speak the target language. The team needs to create a translation back into the consultant’s language.
**Scenario 4:** Translators need to work on Scripture while traveling to remote villages without reliable internet. They want to use their Android tablets.
**Scenario 5:** A Bible translation team has completed drafting the Gospel of Luke. They want to test it with 5 villages (print copies needed).
**Scenario 6:** A team wants to create an oral version of Scripture for non-readers and share it via smartphones.
**Scenario 7:** A team is recording dramatized Scripture and needs to assign speaking parts to voice actors.
**Troubleshooting Scenarios:**
**1.** You get this email: “We’ve been translating Scripture Microsoft Word for 2 years. We have Matthew and Mark completed. Can we import this into Paratext?”
What questions do you ask? What do you tell them about USFM? What challenges might they face?
**2.** A team emails: “We used PTXprint to make a PDF for testing. The community loved it and we’ve made revisions. Now we want to do a professional publication with a Bible society. Do we need to start over?”
How do you respond? What’s the relationship between PTXprint and Publishing Assistant/InDesign?
