# Mastering the Paratext Website

**Estimated time:** 60 minutes

**Purpose:** To learn how to use the [Paratext.org](https://paratext.org/) website in LT work.

## Learning objectives

- You will be able to find information about Paratext using the [Paratext](https://paratext.org/) website.

## Connect

**Why should you use the Paratext.org website?**

Being familiar with the site isn't just a "nice to have" — it's a prerequisite for professional translation work. Here is why:

### 1. Software maintenance and updates

Paratext is constantly evolving. The website is the only official source for:

- **Downloading the latest version:** Using an outdated version can lead to compatibility issues with teammates.
- **Patches and bug fixes:** Translation data is precious; you don't want to be running a version with a known glitch that has already been fixed.
- **System requirements:** Checking if your hardware can handle the latest updates before you click "install."

### 2. Access to critical training resources

Let's be honest: Paratext has a learning curve that feels more like a mountain. The website hosts the **Learning Center**, which includes:

- **Video tutorials:** Visual guides for complex tasks like "Send/Receive" or "Project Check."
- **Manuals and guides:** Searchable documentation that is far more comprehensive than the offline help files.
- **Training courses:** Structured paths to move a user from "Beginner" to "Consultant" level.

### 3. Technical support and troubleshooting

When things go wrong — and in complex software, they eventually do — the website is your lifeline.

- **The support ticket system:** This is where you report bugs or data corruption.
- **Community forums:** Often, another translator halfway across the world has already solved the exact problem you're facing.
- **Knowledge base:** A repository of "how-to" articles that cover common errors.

### 4. Project registration and security

Paratext is not "plug-and-play" in the traditional sense; it requires a high level of **authorization** to protect the integrity of the text.

- **Registration keys:** You must use the website to manage your user registration and ensure your license is active.
- **Project management:** Organizations use the site to manage who has access to which projects, ensuring that only authorized personnel can edit the scripture.

### 5. Plugin and tool integration

The core software is powerful, but many users need specialized tools (like **checklists** or **advanced maps**).

- The website acts as a **marketplace/repository** for plugins that extend what Paratext can do.
- Knowing how to navigate the site allows you to customize your workspace to fit your specific language group's needs.

**The bottom line:** If you aren't comfortable with [paratext.org](http://paratext.org/), you are essentially driving a high-performance vehicle without knowing where the gas station or the mechanic is. Familiarity with the site ensures your data stays **secure**, your software stays **functional**, and your skills stay **current**.

**Question:** A translator in a remote area has limited internet connectivity and speaks Portuguese. They are new to Paratext and need to learn the full translation workflow. Based on the four phases outlined in the Paratext framework, what two specific resources from [Paratext.org](http://paratext.org/) would best support this translator, and why?

The two best resources for this translator are:

- **The Training Manual (Phase 2 – Train).** The training manual is ideal because it requires low bandwidth, making it suitable for areas with limited internet. It functions as a workflow engine that guides the translator through the entire translation cycle — study, draft, revise, check, and publish (stages one through four) — aligning with the chronological project plan. This makes it perfect for a new translator who needs end-to-end learning.
- **Text-based video summaries (Phase 4 – Adapt).** Since streaming tutorial videos requires high bandwidth, Paratext.org provides text-based video summaries as an alternative. These allow the translator to extract essential knowledge without the heavy data payload of streaming video.
- Additionally, because the translator speaks Portuguese, the Language Technology Unit team has expanded training materials into Portuguese as one of the major global languages supported, meaning both of these resources may be available in the translator's language.

**Question:** The Paratext Training Manual is organized alphabetically by software feature to make it easy to look up specific tools. True or false, and why?

- False. The Training Manual is explicitly organized around the *workflow of a real-world translation project*, not alphabetically. It follows project plan stages (Stage 1 through 4), each containing the specific subtasks that need to happen at that point. This process-driven structure means a translator or team leader can follow it sequentially to know exactly what to do next, rather than hunting for isolated features.

**Question:** A translator working in a remote area with limited internet access wants to find out whether a video on "arranging windows" is worth downloading. What feature of [paratext.org](http://paratext.org/) would help them, and what does it provide?

- They should use the **video summary** feature. A video summary is a text-based recap of a specific video lesson. It allows the translator to quickly scan the core topics covered — without downloading the video file itself — so they can decide whether the full video is relevant to their needs. This is especially valuable where bandwidth is slow, costly, or capped.
