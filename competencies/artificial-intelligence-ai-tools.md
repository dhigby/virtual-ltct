# Artificial Intelligence (AI) Tools

**Category:** Technology Domain

## Why it matters

AI tools have become foundational to modern language technology work. This competency enables consultants to amplify their productivity, offload routine cognitive work, sharpen decision-making, and accelerate innovation. Just as importantly, it equips consultants to evaluate AI output critically and to apply AI responsibly in contexts where outputs touch Scripture, sensitive minority-language data, and the trust of partner communities.

## Target competency

> Is able to apply general and specialized AI tools effectively, critically, and responsibly in language technology work, and can guide others in their use.

## Progression by component

_Suggested activities to reach each outcome level._

### 1.0 — General AI Tools (LLMs and AI Assistants)

| Level | Suggested activities |
| --- | --- |
| **Learner** | Can get useful results from a general AI assistant (e.g., Claude, ChatGPT, Gemini, Copilot) for everyday tasks such as drafting, summarizing, brainstorming, and explaining concepts. Understands that AI can hallucinate and that outputs require verification. |
| **Advanced Beginner** | Crafts effective prompts using techniques such as clear context-setting, role assignment, few-shot examples, and structured-output requests. Can compare and choose among major tools for different needs. |
| **Practitioner** | Collaborates with AI assistants as part of regular workflow — using multi-turn refinement, projects/custom instructions, file and image uploads, document analysis, and built-in connectors (Drive, email, calendars, etc.). Knows when AI is *not* the right tool. |
| **Trainer/Proficient** | Trains and coaches others in effective use of general AI tools, including running workshops, producing reference material, and providing one-on-one support. |
| **Expert** | — |

### 2.0 — Specialized Language and Translation AI Tools

| Level | Suggested activities |
| --- | --- |
| **Learner** | Can explain which categories of specialized AI tools apply to which language-work tasks — e.g., neural machine translation, automated quality assessment (AQuA), Greek/Hebrew study tools (Greek Room), automatic speech recognition for low-resource languages (MMS, Whisper), text-to-speech, OCR for non-Latin scripts, alignment, and back-translation tools. |
| **Advanced Beginner** | Uses available specialized AI tools, including SIL-internal tools, for concrete tasks with guidance. |
| **Practitioner** | Uses available specialized AI tools, including SIL-internal tools, for concrete tasks with guidance. |
| **Trainer/Proficient** | Trains and supports users across language teams and domains in adopting specialized AI tools. Provides structured feedback to tool developers and shares best practices across the consultant community |
| **Expert** | — |

### 3.0 — Critical Evaluation and Quality Assurance of AI Output

| Level | Suggested activities |
| --- | --- |
| **Learner** | Recognizes that AI outputs can be wrong, biased, or fabricated — and that this risk is heightened for low-resource languages and for theological, cultural, and named-entity content. |
| **Advanced Beginner** | Routinely verifies AI-generated content against authoritative sources. Identifies common failure modes (hallucinated citations, mistranslated proper nouns, dropped negations, register drift, doctrinal inaccuracy, fluent-but-wrong output) |
| **Practitioner** | Designs and applies systematic review processes for AI-assisted work — spot-checking, back-translation comparison, reference-matching, and human-in-the-loop validation appropriate to the stakes of the task. |
| **Trainer/Proficient** | Trains others in evaluation methods and helps establish quality-assurance standards for AI use within their team or project. |
| **Expert** | — |

### 4.0 — Ethical, Responsible, and Secure Use of AI

| Level | Suggested activities |
| --- | --- |
| **Learner** | Understands that data submitted to AI tools may be stored, used for model training, or exposed — and that unpublished translations, partner data, and community language data require special care. Aware of relevant SIL and Wycliffe AI guidelines. |
| **Advanced Beginner** | Applies basic data-handling practices: redacting sensitive content, choosing tools with appropriate privacy/retention settings, respecting community ownership of language data, and disclosing AI-assisted work appropriately. |
| **Practitioner** | Evaluates AI tools for fitness in a given context — weighing privacy, licensing, cost, cultural appropriateness, and theological reliability — and advises colleagues on tool selection. |
| **Trainer/Proficient** | Trains and advises others on responsible AI practice, and helps shape local guidelines for AI use with the language communities they serve. |
| **Expert** | — |

### 5.0 — AI Workflow Integration and Automation

| Level | Suggested activities |
| --- | --- |
| **Learner** | Understands that AI can be embedded into larger workflows beyond chat interfaces — through APIs, agents, scripts, IDE integrations, custom assistants (Projects, Custom GPTs, Gems), and connectors to tools like Drive, email, and code repositories. |
| **Advanced Beginner** | Uses pre-built AI integrations to streamline routine work — for example, AI features inside Office, VS Code, or browsers; configured Projects/Custom GPTs with system prompts and reference files; AI-assisted search across personal documents. |
| **Practitioner** | Builds simple AI-augmented automations or lightweight agents to handle recurring language-tech tasks — for example, using Claude Code or Copilot to generate and run scripts, chaining steps with tools like n8n or Zapier, or producing small custom assistants tuned to a specific consulting workflow. |
| **Trainer/Proficient** | Designs and shares reusable AI-augmented workflows, scripts, or agents with the wider consultant community. Mentors others in safely extending their own work with automation. |
| **Expert** | — |
