# Installing an Open-Source Text as a Resource Project

**Estimated time:** 35 minutes

**Purpose:** Prepares you to give a translation team — or a training workshop — a public-domain
or openly-licensed Scripture text they can actually edit, by installing it as its own standard
translation project rather than as a locked Paratext resource.

## Learning objectives

- You will be able to install an openly-licensed/open-source Scripture text as a readable
  project in Paratext, distinct from the team's own translation draft project.

## Connect

**✏️ Reflection:** Think about a translation team you support, or a training workshop you have
run or attended. Has a team ever asked you for a text they could freely practice editing on —
without any risk to their own translation draft? Or has a workshop needed every participant to
be able to edit the *same* text at the same time, so everyone could try out Paratext features
together?

Hold that situation in mind as you work through this lesson — it is exactly the case this
lesson prepares you for.

## Content

**Key point: Paratext resources are never editable — even when the underlying text is public
domain.**

A team might download a well-known open text, such as the Berean Standard Bible (BSB), and be
surprised that they still can't type into it. That surprise is the most common failure point
consultants see with this task: users assume that because a text is public domain, Paratext
will let them edit it directly. It won't. Paratext resources (the read-only reference texts
listed alongside a project) are locked regardless of the license on the underlying text. To get
an editable copy, you have to bring the text in as its own **project**, not as a resource.

**The procedure**

1. **Find the text in USFM format, outside Paratext.** Paratext does not fetch these for you —
   you need to locate and download a USFM edition yourself. Two places to look:
   - [ebible.org/find](https://ebible.org/find)
   - [open.bible/bibles](https://open.bible/bibles)
2. **Create a new project in Paratext** (main menu → New Project).
3. **Specify the language** in the project's properties.
4. **Set the project type to a standard translation project** — not a resource.
5. **Import the books** from your downloaded USFM files (Project menu → Manage books → Import
   books).

Once imported this way, the text lives inside an ordinary, editable Paratext project — separate
from the team's own translation draft — that anyone with access can type into, mark up, or
practice on without touching the team's real work.

**INFO:** This applies to any Paratext 9 version, and the same principles apply to earlier
Paratext versions too — the distinction between a locked resource and an editable project has
not changed across versions.

### Key Takeaways

- A Paratext **resource** is always read-only, no matter what license the text carries.
- To get an *editable* copy of an open-license text, download it in USFM format from outside
  Paratext (e.g., ebible.org/find, open.bible/bibles), then create a new **standard translation
  project** and import the books into it.
- This is the fix for the common confusion of "why can't I edit this public-domain text?" — the
  text isn't broken, it's a resource; the team needs it as a project instead.

## Challenge

**✏️ Try this:** A translator on a team you support wants a text to practice editing on before
they touch their team's real draft — they've heard that the Berean Standard Bible is public
domain and assumed they could just open it and start typing.

Write out, as if explaining it directly to this translator:

1. Why they cannot currently edit the BSB the way it's set up now.
2. Where they could get a USFM copy of it (or another open-license text you might recommend for
   a similar case).
3. The exact sequence of steps you would walk them through in Paratext to turn that download
   into a text they can freely edit.

Also consider the workshop version of this same task: if you were setting up a room of ten
participants who all need to edit the *same* open text during a training session, does anything
about the steps above change? (Think about what "the same text" means when each participant
creates their own project.)
