# Parallel passages - review

This is a solid, high-level workflow for Paratext users. It correctly identifies the tension in parallel passage checking: balancing **textual consistency** with **authorial intent**.
Based on the module text provided, here is my review and suggested improvements to make it more precise for an "Experienced Translator" audience.
---
## 1. Technical Accuracy & Precision
- **The Menu Path:** In Part 1, you list `≡ Tab > Tools > Parallel Passages`. In recent versions of P~~aratext (9.x+), the Parallel Passages tool is often accessed via the **Checking** menu ~~or the **Project Plan**. ~~If your LMS is targeting Paratext 9.4+, I suggest adding: *"If not in Tools, l~~ook under your **Project Plan** tasks."*
- **The "Source" Language:** The module mentions reviewing consistency, but it doesn't explicitly mention the **Source Language tool**.
	- *Suggestion:* Add a note that translators should use the **Source Language** text (Greek/Hebrew) within the Parallel Passages tool to verify if the source is actually identical before they try to harmonize the translation.
## 2. Structural Improvements
- **The "Inventory" Workflow:** Part 2 mentions updating inventories. This is the most time-consuming part of Paratext checking.
	- *Suggestion:* Explicitly state that **Inventories must be "Send/Received"** to be shared. A common point of failure in teams is one person marking things "Valid" in an inventory, but the teammate still seeing them as "Errors" because the inventory wasn't synced.
- **Parallel Passages Filter:** The module mentions the basic process but skips a key time-saver.
	- *Suggestion:* Mention the **Filter** settings in the Parallel Passages tool. Users can filter for "Only differences" or "All passages." For a 60-minute module, teaching them to filter for "Only differences" is a high-value tip.
## 3. Critical Additions (Missing from the Module)
The following items are not in your text but are vital for a "Comprehensive" workflow:
- **The "Compare Texts" Tool:** While the module focuses on the Parallel Passages tool, experienced users often use the **Compare Texts** feature to look at their project against a Model translation (like the NIV or a regional lingua franca) to see how others handled the parallel.
- **Note on Footnotes:** Parallel passages often require cross-references.
	- *Question:* Does this module intend to cover **Cross-Reference** checks? If so, it should mention the **Project Settings \> Biblical Terms/Parallel Passages** linkage that helps generate cross-references automatically.
- **The "Ignore" vs. "Valid" distinction:** In the Basic Checks section, you mention "Some issues may be intentional (and can be ignored)."
	- *Refinement:* In Paratext, "Ignoring" an error is different from marking a character or punctuation as "Valid" in an Inventory. I suggest clarifying that **Validating in the Inventory** is the "Gold Standard" because it clears the error for the whole team, whereas "Ignoring" is often just local/temporary.
## 4. Suggested "Pro-Tips" for the Level: Experienced
Since this is for "Experienced" users, you might add these two bullets to Part 2:
- **Regex Checks:** Mention that "Custom Checks" (Regular Expressions) can be used for complex parallel patterns that the standard tool might miss.
- **Checklist Status:** Remind users that marking a check as "Done" in the **Assignments and Progress** window is what triggers the "Green Checkmark" for the Consultant’s view.

To ensure I don't "hallucinate" features that aren't intended for this specific training:
1. Does your specific LMS setup include the **Project Plan**? (If yes, the instructions should point them to the Plan rather than the Tools menu).
