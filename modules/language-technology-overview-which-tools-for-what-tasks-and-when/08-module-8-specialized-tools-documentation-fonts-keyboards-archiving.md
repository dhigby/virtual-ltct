#  Module 8: Specialized Tools - Documentation, Fonts, Keyboards & Archiving

## Connect (5 minutes)
**Reflection:** Beyond the core work of translation and literacy, language projects need infrastructure—proper documentation, fonts that display correctly, keyboards for typing special characters, and secure archiving.
These “supporting” tools are often invisible until something breaks. Then they become critical.
### Content (25 minutes)
Play the video Other Specialized LT Tools.
### **Specialized Anthropology & Data Tools**
**Notebook (part of FLEx)** - Anthropological data
- Tracks and analyzes anthropological findings
- Integrated with linguistic work
**SILKin** - Kinship terminology
- Facilitates collection, analysis, and reporting of kinship terms
- Specialized cultural documentation
**RAMP & REAP** - Archiving applications
- Package metadata and resources
- Submission to institutional repositories
- Long-term data preservation
### **Language Learning Tools**
**Vocab Manager** - Flashcard-based learning
- Systematic vocabulary acquisition
**Anki** - Flashcard system
- Also used for language learning
- Spaced repetition methodology
### **Documentation Tools**
**SayMore** - Language documentation project management
- Manages raw linguistic data for language description
- Two categories: documented events and native speakers
- Collects and organizes information with metadata
- Preserves documentation systematically
**Encoding & Font Tools**
**SIL Converters** - Encoding conversion
- Changes encoding, font, and/or script
- Works with Word, LibreOffice, XML, SFM documents
- System-wide encoding converter repository
- Uses TECkit maps and CC tables
**Unicode Fonts:**
- **Doulos SIL** - Comprehensive Roman/Cyrillic characters (including IPA)
- **Charis SIL** - Serif, optimized for long documents, full style set (regular, italic, bold, bold italic)
- **Andika** - Designed specifically for literacy materials
- **Gentium** - For diverse ethnic groups using Latin, Cyrillic, Greek scripts
### **Keyboard Tools**
**Keyman** - Cross-platform keyboard solution
- Create and use keyboard layouts for any language
- Catalog of existing layouts
- Tools for creating custom layouts
- Works on all major platforms
**Microsoft Keyboard Layout Creator** - Windows keyboard creation
- Creates layouts for Windows
- For languages Microsoft doesn’t support
### Challenge (25 minutes)
**Scenario-Based Problem Solving:**
**Scenario 1 - Font Problems:** A translator emails: “When I type certain characters, they show up as boxes. But my colleague’s computer displays them fine. We’re both using Word.”
What’s likely happening? What tool or solution would you recommend?
**Scenario 2 - Documentation:** A linguistic team has been collecting stories, songs, and interviews for 3 years. Everything is scattered across folders with inconsistent file naming. They’re losing track of speakers and events.
What tool should they have been using from the start? What can they do now to organize this data?
**Scenario 3 - Keyboard Creation:** A language uses 5 special characters not found on standard keyboards. Community members need to type these on phones, computers, and tablets.
What tool addresses this need across all platforms?
**Scenario 4 - Archiving:** A long-term project is wrapping up. They have 15 years of data: recordings, transcriptions, lexicons, grammars, and published materials. How should they preserve this for future generations?
What tools would you recommend for proper archiving?
### Change (10 minutes)
**Self-Assessment:**
- Can you explain the difference between fonts and keyboards?
- What are the key archiving and documentation tools, and why do they matter long-term?
- Why is proper documentation important even when it’s not directly producing translation output?
