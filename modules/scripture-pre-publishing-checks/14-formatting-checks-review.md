# Formatting checks (?review??)

**Module 6: Formatting Checks**<br>**Duration:** 60 minutes<br>**Level:** Experienced Translator<br>**Goal:** Complete comprehensive formatting verification using Paratext checklists to ensure publication-ready text.**<br>Procedure Walkthrough**<br>*Time: 45 minutes***<br>**
**Step 1: Structural Foundation (Basic Checks)**<br>`≡ Tab > Tools > Basic Checks`<br>Before running any formatting checklists, you must verify the structural "anchors" of the text. If these are broken, other checklists will give false or confusing results.<br>• **Chapter/Verse Sequence:** Ensures no numbers are skipped or out of order.<br>• **Duplicate Verses:** Flags if a verse number appears twice.
💡 **Note:** If a chapter marker (`\c`) is missing, Paratext will report that the verses in the following section are **duplicated** because it thinks they still belong to the previous chapter. Resolve these first.<br>• **Marker Pairs:** Use this to catch unclosed character styles (like `\nd`...`\nd*`).**<br>**
**Step 2: Marker Count & Pair Inference**<br>`≡ Tab > Tools > Checklists > Markers`<br>The Markers checklist provides a "census" of every USFM marker in your project. It does not flag errors for you; you must **infer** them from the totals.<br>• **Manual Pair Audit:** For markers that require an opener and a closer (like `\f` and `\f*`), the counts **must match exactly**.<br>    ◦ *Example:* If you see 412 instances of `\f` but only 410 instances of `\f*`, you have two unclosed footnotes that will break the publication layout.<br>• **Anomalies:** Look for "typo" markers. If the list shows `\p` used 500 times but `\q` used only once, check if that single `\q` was a mistake.
**Step 3: Long and Short Verses**<br>`≡ Tab > Tools > Checklists > Long/short verses`<br>Once the sequence is solid, use this to find content anomalies.<br>• **Flagged as too long:** Check for missing verse breaks or accidental text duplication.<br>• **Flagged as too short:** Check for accidental deletions.<br>• **Action:** There is no "Correct" flag to click in this checklist. You must manually investigate each flagged verse. If the verse is legitimately long or short in the original, simply move to the next item.**<br>**
**Step 4: Word or Phrase Checklist**<br>`≡ Tab > Tools > Checklists > Word or phrase`<br>This is a powerful "Search and Destroy" tool for final verification.<br>• **Consistency Check:** Find all instances of a specific word to ensure you haven't used multiple spellings.<br>• **Specific Term Audit:** While the **Biblical Terms Tool** is the primary way to check key terms, this checklist is a fast way to see every occurrence of a specific phrase in context to ensure late-stage edits didn't introduce inconsistencies.**<br>**
**Step 5: Navigation (Headings and Titles)**<br>`≡ Tab > Tools > Checklists > Section headings`<br>`≡ Tab > Tools > Checklists > Book titles`<br>• **Headings:** Check for "Heading Deserts"—long stretches of text without a break. Ensure `\s1` and `\s2` follow your project's hierarchy.<br>• **Titles:** Verify the Header (`\h`) and Main Title (`\mt1`) against the official approved list. **Typing errors here are high-visibility mistakes.<br>**
**Step 6: References and Footnotes**<br>`≡ Tab > Tools > Checklists > References`<br>`≡ Tab > Tools > Checklists > Footnotes`<br>• **Reference Validity (Red Icons):** If your markers are correct, Paratext will display a **red icon** next to the reference if the book abbreviation is wrong or the chapter/verse number doesn't exist in the project.<br>• **Footnote Callers:** Using the `\f +` marker is best practice, as it allows the publishing software to automatically generate consistent, sequential callers (a, b, c...).<br>• **Footnote Origin (****`\fr`****):** These are usually inserted automatically, but if a translator manually edited a footnote to fix a mistake, the `\fr` might point to the wrong verse. Use the checklist to verify the footnote content matches the verse location.**<br>Troubleshooting & Tips<br>Problem Solution Red icon in references<br>**Check for typos in book names (e.g., `Mat` vs `Matt`) or ensure the reference actually exists in the Bible.<br>**Marker counts don't match<br>**Use the **Markers** checklist to find the book/chapter with the discrepancy and manually find the missing `*` closer.<br>**Missing headings<br>**Check your project's Style Guide. Some projects require a heading at the start of every chapter; others only at major story breaks.<br>**Next Steps:** Once these structural and formatting checks are clean and all red icons have been addressed, you are ready for **Module 7: Glossary Linking.**
